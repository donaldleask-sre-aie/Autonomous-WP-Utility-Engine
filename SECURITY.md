The Autonomous-WP-Utility-Engine is a powerful tool designed to execute privileged commands (database access, file modification) within a WordPress environment. The security of this agent is paramount.

If you believe you have discovered a vulnerability, we ask that you adhere to responsible disclosure practices outlined below.

1. Responsible Disclosure Policy
DO NOT publicly post vulnerabilities (e.g., in a blog post, social media, or the public GitHub Issues tab) before contacting the author and allowing a reasonable time for a patch.

DO NOT attempt to exploit a vulnerability against live production systems or other users' installations.

DO NOT disclose the vulnerability until it has been acknowledged and a patch has been released by the project author.

2. How to Report a Vulnerability
All security-related reports must be sent via encrypted email to the following address:

Email: security.agent@dleaskdev.com (Placeholder: Use a dedicated, non-ARAGS security email address you control)

Please provide the following information in your report:

Vulnerability Description: A clear explanation of the flaw and its potential impact.

Reproduction Steps: Detailed steps necessary to reproduce the vulnerability.

Proof of Concept (PoC): Any code or script necessary to demonstrate the exploit.

Affected Version: The version or commit hash of the code where the flaw exists.

3. Timeline and Response
We commit to the following response timeline:

Acknowledgement: You will receive a confirmation email within 48 hours of submission.

Assessment: We will perform a review and provide an estimated resolution timeline within 7 days.

Credit: After the patch is released and verified, you will be publicly credited in the release notes unless you request anonymity.

Your adherence to this responsible disclosure process is greatly appreciated and ensures the safety of the entire community using this project.
